# 标签不显示问题诊断指南

## 问题现象
列表页和详情页的文章标题前没有显示彩色标签（如"紧急预警"、"案例分析"等）。

## 可能的原因

### 1. 数据问题
**检查方法：**
1. 打开微信开发者工具
2. 点击"控制台"标签
3. 刷新页面，查看日志输出

**应该看到的日志：**
```
========== 页面加载 ==========
本地文章数据示例： {id: 1, tag: "紧急预警", tagType: "danger", ...}
========== 开始加载文章 ==========
格式化文章： {id: "xxx", title: "xxx", tag: "紧急预警", tagType: "danger"}
格式化后的文章列表： [{tag: "紧急预警", tagType: "danger", ...}, ...]
```

**如果没有 tag 和 tagType：**
- 说明云数据库中的文章缺少这些字段
- 需要更新数据库数据

### 2. 样式问题
**检查方法：**
1. 在微信开发者工具中，点击"调试器"
2. 选择一个文章卡片元素
3. 查看是否有 `.tag` 类名
4. 查看样式是否正确应用

**应该看到的 HTML 结构：**
```html
<view class="news-card">
  <view class="tag tag-danger">紧急预警</view>
  <view class="news-title">警惕！"刷单兼职"骗局...</view>
  ...
</view>
```

**应该看到的样式：**
```css
.tag {
  display: inline-block;
  padding: 8rpx 24rpx;
  border-radius: 8rpx;
  font-size: 22rpx;
  ...
}

.tag-danger {
  background: #fee2e2;
  color: #dc2626;
}
```

### 3. 编译缓存问题
**解决方法：**
1. 在微信开发者工具中，点击"清缓存" → "清除文件缓存"
2. 点击"清缓存" → "清除数据缓存"
3. 点击"编译"按钮重新编译

### 4. 云数据库数据缺失
**检查方法：**
1. 打开微信开发者工具
2. 点击"云开发"按钮
3. 进入"数据库"
4. 选择 `articles` 集合
5. 查看数据是否包含 `tag` 和 `tagType` 字段

**如果缺少字段，需要更新数据：**

#### 方法1：手动更新（适合少量数据）
在云开发控制台，编辑每条记录，添加：
```json
{
  "tag": "紧急预警",
  "tagType": "danger"
}
```

#### 方法2：批量更新（推荐）
创建一个临时云函数来批量更新：

```javascript
// cloudfunctions/updateArticleTags/index.js
const cloud = require('wx-server-sdk')
cloud.init({ env: cloud.DYNAMIC_CURRENT_ENV })
const db = cloud.database()

exports.main = async (event, context) => {
  try {
    // 获取所有文章
    const { data: articles } = await db.collection('articles').get()
    
    // 批量更新
    const tasks = articles.map(article => {
      // 根据标题或内容判断类型
      let tag = '知识科普'
      let tagType = 'info'
      
      if (article.title.includes('警惕') || article.title.includes('预警')) {
        tag = '紧急预警'
        tagType = 'danger'
      } else if (article.title.includes('案例') || article.title.includes('揭秘')) {
        tag = '案例分析'
        tagType = 'warning'
      } else if (article.title.includes('防骗') || article.title.includes('陷阱')) {
        tag = '防骗知识'
        tagType = 'warning'
      }
      
      return db.collection('articles').doc(article._id).update({
        data: { tag, tagType }
      })
    })
    
    await Promise.all(tasks)
    
    return {
      success: true,
      message: `已更新 ${articles.length} 篇文章`
    }
  } catch (err) {
    return {
      success: false,
      error: err.message
    }
  }
}
```

然后在云开发控制台测试运行这个云函数。

## 临时解决方案：启用调试信息

如果标签仍然不显示，可以临时启用调试信息来查看数据：

1. 打开 `pages/index/index.wxml`
2. 找到这一行（约第 80 行）：
```xml
<!-- <view style="font-size: 20rpx; color: #999;">tag: {{item.tag}}, type: {{item.tagType}}</view> -->
```
3. 删除注释符号，改为：
```xml
<view style="font-size: 20rpx; color: #999;">tag: {{item.tag}}, type: {{item.tagType}}</view>
```
4. 保存并编译

这样会在每个文章卡片上显示 tag 和 tagType 的值，帮助你诊断问题。

## 快速测试：使用本地数据

如果云数据库有问题，可以先使用本地数据测试：

1. 打开 `pages/index/index.js`
2. 找到 `onLoad` 方法（约第 180 行）
3. 临时修改为：
```javascript
onLoad(options) {
```

## 验证步骤

完成上述修改后，按以下步骤验证：

1. **清除缓存**
   - 清除文件缓存
   - 清除数据缓存

2. **重新编译**
   - 点击"编译"按钮

3. **查看控制台**
   - 应该看到文章数据包含 tag 和 tagType

4. **查看页面**
   - 列表页每个文章卡片上方应该有彩色标签
   - 点击进入详情页，标题上方也应该有标签

5. **检查样式**
   - 紧急预警：淡红色背景
   - 案例分析/防骗知识：淡黄色背景
   - 知识科普：淡蓝色背景

## 常见问题

### Q1: 标签显示为空白
**A:** 检查 `{{item.tag}}` 是否有值，可能是数据中 tag 字段为空字符串。

### Q2: 标签没有颜色
**A:** 检查 `{{item.tagType}}` 是否正确，应该是 danger/warning/info 之一。

### Q3: 标签样式不对
**A:** 检查 wxss 文件是否正确保存，尝试清除缓存重新编译。

### Q4: 详情页没有标签
**A:** 检查 `pages/article-detail/article-detail.js` 中的数据格式化逻辑，确保包含 tag 和 tagType。
