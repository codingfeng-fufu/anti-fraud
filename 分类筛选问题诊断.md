# 分类筛选功能问题诊断与修复

## 问题现象
点击顶部的诈骗类型选项卡后，下方资讯列表显示为空，无法显示对应类别的文章。

## 已完成的修复

### 1. 云函数优化
**文件：** `cloudfunctions/getArticles/index.js`

**修改内容：**
- ✅ 添加详细的日志输出，便于调试
- ✅ 将排序字段从 `publishTime` 改为 `timestamp`（因为数据库中使用的是 timestamp）
- ✅ 优化查询条件构建逻辑

**关键代码：**
```javascript
// 分类筛选
if (category && category !== '全部') {
  query.category = category
  console.log('添加分类筛选：', category)
}

// 使用 timestamp 排序
const result = await db.collection('articles')
  .where(query)
  .orderBy('timestamp', 'desc')
  .skip((page - 1) * pageSize)
  .limit(pageSize)
  .get()
```

### 2. 前端逻辑优化
**文件：** `pages/index/index.js`

**修改内容：**
- ✅ 在 `selectCategory` 方法中添加详细日志
- ✅ 确保切换分类时正确重置状态（currentPage、articles、hasMore）

**关键代码：**
```javascript
selectCategory(e) {
  const category = e.currentTarget.dataset.category
  
  console.log('========== 切换分类 ==========')
  console.log('选中分类：', category)
  
  this.setData({
    selectedCategory: category,
    currentPage: 1,      // 重置为第1页
    articles: [],        // 清空现有列表
    hasMore: true        // 重置加载状态
  })
  
  // 重新加载文章
  this.loadArticlesFromCloud(true)
}
```

### 3. 空状态优化
**文件：** `pages/index/index.wxml` 和 `pages/index/index.wxss`

**修改内容：**
- ✅ 添加空状态提示（"暂无相关资讯"）
- ✅ 区分"没有更多数据"和"该分类无数据"两种情况

### 4. 批量更新分类工具
**文件：** `cloudfunctions/updateArticleCategories/`

创建了新的云函数，用于批量更新数据库中文章的 category 字段。

## 可能的问题原因

### 原因1：数据库文章缺少 category 字段 ⚠️
**检查方法：**
1. 打开云开发控制台
2. 进入数据库 -> articles 集合
3. 查看文章数据是否有 `category` 字段

**解决方案：**
运行 `updateArticleCategories` 云函数批量添加分类字段。

### 原因2：category 字段值不匹配
**检查方法：**
查看数据库中的 category 值是否与前端分类名称完全一致。

**前端分类列表：**
```javascript
categories: [
  '全部', 
  '刷单返利', 
  '虚假投资理财', 
  '虚假购物服务', 
  '冒充电商客服',
  '虚假贷款', 
  '虚假征信', 
  '冒充领导熟人', 
  '冒充公检法', 
  '网络婚恋交友', 
  '网游虚假交易'
]
```

**解决方案：**
确保数据库中的 category 字段值与上述列表完全一致（包括大小写、标点符号）。

### 原因3：排序字段不存在
**问题：**
原代码使用 `publishTime` 排序，但数据库中可能没有这个字段。

**解决方案：**
已修改为使用 `timestamp` 字段排序。

## 常见问题

### Q1: 切换分类后一直显示"正在加载..."
**A:** 检查云函数是否正确部署，查看云函数日志是否有错误。

### Q2: 所有分类都显示相同的文章
**A:** 检查云函数的查询条件是否正确，确认 category 参数是否正确传递。

### Q3: 某些分类有数据，某些没有
**A:** 这是正常的，说明数据库中该分类确实没有文章。可以手动添加或等待内容更新。

### Q4: "全部"分类也显示为空
**A:** 检查数据库中是否有文章数据，确认 timestamp 字段是否存在。
