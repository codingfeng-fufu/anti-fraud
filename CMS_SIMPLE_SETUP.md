# CloudBase CMS 简化配置指南

## 🎯 如果找不到特定的字段类型

CloudBase CMS 的界面可能因版本不同而有差异。下面提供**简化版配置**，使用最基本的字段类型。

---

## 📝 简化版字段配置

### 最小化必须字段（只用 4 种字段类型）

如果你在 CMS 中只看到这些字段类型：
- ✅ 单行文本（String / Text）
- ✅ 多行文本（Textarea）
- ✅ 富文本（Rich Text）
- ✅ 图片（Image）
- ✅ 数字（Number）
- ✅ 日期时间（DateTime / Date）

那么使用以下简化配置：

---

### 1. 标题（title）
```
字段名：title
类型：单行文本
必填：是
```

### 2. 分类（category）
```
字段名：category
类型：单行文本
必填：是
默认值：大学生

使用时手动输入分类：
- 刷单诈骗
- 校园贷
- 电信诈骗
- 网购退款
- 杀猪盘
- 投资理财
- 大学生
```

### 3. 标签（tag）
```
字段名：tag
类型：单行文本
必填：是
默认值：知识科普

使用时手动输入标签：
- 紧急预警
- 案例分析
- 防骗知识
- 知识科普
- 视频教程
```

### 4. 标签类型（tagType）
```
字段名：tagType
类型：单行文本
必填：是
默认值：info

使用时手动输入：
- danger（红色）
- warning（橙色）
- info（蓝色）
```

### 5. 封面图（coverImage）
```
字段名：coverImage
类型：图片（Image）
必填：否
```

### 6. 正文（content）
```
字段名：content
类型：富文本（Rich Text）✅ 关键字段
必填：是
```

### 7. 作者（author）
```
字段名：author
类型：单行文本
必填：否
默认值：反诈小助手
```

### 8. 浏览量（views）
```
字段名：views
类型：数字（Number）
必填：否
默认值：0
```

### 9. 点赞数（likes）
```
字段名：likes
类型：数字（Number）
必填：否
默认值：0
```

### 10. 时间戳（timestamp）
```
字段名：timestamp
类型：日期时间（DateTime）
必填：是
默认值：当前时间
```

### 11. 状态（status）
```
字段名：status
类型：单行文本
必填：是
默认值：draft

使用时手动输入：
- draft（草稿）
- published（已发布）
```

## 🚀 快速创建文章示例

### 填写示例

当你创建新文章时，这样填写：

```

（手动输入，表示已发布）
```

## 🎨 富文本编辑器

这是最重要的字段类型！

### 功能

```
工具栏：
【B】加粗
【I】斜体  
【U】下划线
【H】标题（H1-H6）
【•】列表（有序/无序）
【🔗】插入链接
【📷】插入图片 ✅
【""】引用
【≡】对齐方式
```

### 使用技巧

**插入图片：**
1. 点击图片图标
2. 上传图片
3. 图片自动上传到云存储
4. 自动插入到文章中

**格式化文本：**
1. 选中文字
2. 点击工具栏按钮
3. 即时生效

## 📊 数据库集合名称

确保创建的内容类型标识为：`articles`

```
内容模型配置：
名称：文章
标识：articles  ✅ 必须是这个名称
描述：反诈知识文章
```

小程序代码中查询的就是这个集合：
```javascript
db.collection('articles')  // 必须匹配
```

## ✅ 最小可用配置

如果时间紧张，只需要这 5 个核心字段就能工作：

### 核心字段（最少）

1. **title** - 单行文本（标题）
2. **content** - 富文本（正文）✅

4. **status** - 单行文本（draft 或 published）

## 🔍 常见问题

### Q: 找不到"富文本"类型？

**可能的名称：**
- Rich Text
- HTML Editor
- WYSIWYG Editor
- Markdown Editor

**如果实在找不到：**
使用"多行文本（Textarea）"代替，但不支持格式化。

### Q: 字段名能用中文吗？

**不建议！**
- 显示名称：可以用中文（如"标题"）
- 字段标识：必须用英文（如 `title`）

### Q: 必须添加所有 11 个字段吗？

**不必须！**

最小配置只需要 5 个核心字段：
- title
- content
- timestamp
- status
- views

其他字段可以后续添加。

### Q: 如何确认字段名是否正确？

创建内容类型后，在字段列表中会显示：
```
显示名称：标题
字段标识：title  ← 这个必须正确
```

## 💡 实在不行的替代方案

如果 CloudBase CMS 配置太复杂，可以：

### 方案 1：直接在云数据库中添加

```
1. 云开发控制台 → 数据库
2. 创建 articles 集合
3. 点击「添加记录」
4. 手动填写 JSON 数据：

{
  "title": "文章标题",
  "category": "刷单诈骗",
  "tag": "紧急预警",
  "tagType": "danger",
  "content": "<p>文章内容...</p>",
  "coverImage": "",
  "author": "反诈小助手",
  "views": 0,
  "likes": 0,
  "timestamp": 1736425200000,
  "status": "published"
}
```

### 方案 2：使用云函数批量导入

创建一个云函数，批量导入初始文章。
