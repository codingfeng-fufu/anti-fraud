# 快速修复分类筛选问题

## 问题确认
除了"全部"标签外，其他分类标签都显示"暂无相关资讯"。

## 原因
数据库中的文章缺少 `category` 字段。

## 解决方案

### 方案1：运行云函数批量更新（推荐）⭐

#### 步骤1：部署云函数
1. 在微信开发者工具中，找到 `cloudfunctions/updateArticleCategories` 文件夹
2. 右键点击该文件夹
3. 选择"上传并部署：云端安装依赖"
4. 等待部署完成（约30秒-1分钟）

#### 步骤2：运行云函数
1. 打开微信开发者工具
2. 点击顶部的"云开发"按钮
3. 在云开发控制台中，点击左侧"云函数"
4. 找到 `updateArticleCategories` 云函数
5. 点击该云函数进入详情页
6. 点击"测试"标签
7. 在"参数"输入框中输入：`{}`
8. 点击"运行测试"按钮
9. 等待执行完成，查看返回结果

**预期返回结果：**
```json
{
  "success": true,
  "message": "成功更新 XX 篇文章的分类",
  "count": XX
}
```

#### 步骤3：验证修复
1. 回到小程序
2. 清除缓存（工具栏 → 清缓存 → 清除文件缓存 + 清除数据缓存）
3. 重新编译
4. 点击不同的分类标签
5. 应该能看到对应分类的文章了

### 方案2：手动更新数据库

如果方案1不可行，可以手动更新：

#### 步骤1：打开数据库
1. 点击"云开发"按钮
2. 点击左侧"数据库"
3. 选择 `articles` 集合

#### 步骤2：检查数据
随机点开几条文章记录，查看是否有 `category` 字段。

#### 步骤3：手动添加字段
对于每条记录：
1. 点击"编辑"
2. 添加字段 `category`
3. 根据文章标题选择合适的分类值：
   - 刷单返利
   - 虚假投资理财
   - 虚假购物服务
   - 冒充电商客服
   - 虚假贷款
   - 虚假征信
   - 冒充领导熟人
   - 冒充公检法
   - 网络婚恋交友
   - 网游虚假交易

**示例：**
```json
{
  "_id": "xxx",
  "title": "警惕！"刷单兼职"骗局又出新花样",
  "category": "刷单返利",  ← 添加这个字段
  "tag": "紧急预警",
  "tagType": "danger",
  ...
}
```

## 常见问题

### Q: 运行云函数后仍然没有数据？
**A:** 
1. 检查云函数是否执行成功（查看返回结果）
2. 清除小程序缓存并重新编译
3. 检查数据库中的文章是否真的更新了 category 字段

### Q: 某些分类有数据，某些没有？
**A:** 
这是正常的。`updateArticleCategories` 根据标题关键词自动分类，如果标题中没有相关关键词，可能会被分到"全部"或其他分类。可以手动调整。

### Q: 所有分类都显示相同的文章？
**A:** 
检查云函数的查询逻辑，确认 category 参数是否正确传递。查看云函数日志。

### Q: 云函数部署失败？
**A:** 
1. 检查网络连接
2. 确认云开发环境已开通
3. 查看错误信息，可能是依赖安装失败
